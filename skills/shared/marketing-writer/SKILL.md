# Marketing Writer

Write authentic marketing content that connects with real people. This skill reads your codebase to automatically understand your product, then writes copy that sounds like a real conversation — not a press release.

## How It Works

When invoked:

1. **Read the codebase first** to understand:
 - What the product does
 - Key features and how they work
 - The core value proposition
 - Who this solves problems for

2. **Match the tone**: casual, direct, benefit-focused, no corporate speak

3. **Use the right format** based on what's needed:
 - **Landing Page Sections**: Problem → Solution → Benefit
 - **Tweet Threads**: Hook → Build Credibility → Show Value → CTA
 - **Launch Emails**: Personal, specific value prop, easy CTA

4. **Write content that**:
 - Focuses on real benefits, not features for features' sake
 - Uses simple, direct language
 - Avoids jargon and buzzwords
 - Explains WHY the feature matters to the user
 - Actually converts

## How to Use This Skill

**For a new feature launch:**
"I just shipped [describe the feature]. Write me marketing copy."

The skill reads your code, understands what you built, and generates landing page copy, tweets, and an email.

**For a specific content type:**
- "Write a tweet thread about [feature]"
- "Create a landing page section for our [feature]"
- "Draft a launch email"

## What Gets Read Automatically

- Core app files (README, package.json, requirements.txt)
- Feature implementations
- Documentation
- Any existing marketing copy for tone reference

## Brand Voice Principles

- Talk like a human, not a brand
- Lead with the benefit, not the feature
- Short sentences. Active voice.
- If a 10-year-old wouldn't understand it, rewrite it
- No: "leverage," "synergy," "cutting-edge," "seamless," "empower"
- Yes: plain words that say exactly what they mean

## Content Templates

### Landing Page Section
```
[Problem statement — make them nod]
[How your product solves it — be specific]
[What they get — concrete benefit]
[CTA — one thing, make it obvious]
```

### Tweet Thread
```
Tweet 1: Hook — surprising fact, bold claim, or relatable pain
Tweet 2-4: Build the case — specifics, proof, story
Tweet 5-6: Show the value — what changes for the reader
Tweet 7: CTA — what to do next
```

### Launch Email
```
Subject: [Specific, not clever]
Opening: One sentence about the problem
Body: What you built and why it matters to them
Proof: One concrete example or result
CTA: One link, one action
```
